## 1. 프로젝트 목적

### 왜 만드는가

PC 구매는 부품 용어, 호환성, 가격 변동, 조립 과정 때문에 일반 고객에게 진입 장벽이 높습니다. JHSComputer2는 고객이 복잡한 부품 지식을 몰라도 목적에 맞는 PC를 쉽게 고르고 주문할 수 있게 만드는 것을 목표로 합니다.

### 해결하려는 문제

- 고객은 어떤 CPU/GPU/메인보드를 골라야 하는지 판단하기 어렵다.
- 게임별 성능과 실제 체감 기준을 견적 단계에서 알기 어렵다.
- 온라인 최저가와 실제 조립/배송 가능한 구성 사이에 차이가 있다.
- 주문 후 진행 상태를 명확하게 확인하기 어렵다.
- 운영자는 견적, 주문, 상태 변경, 추천글 관리를 한 곳에서 보기 어렵다.

### 서비스 핵심 방향

- 단순 부품 쇼핑몰이 아니라 `견적 추천 -> 전문가 검수 -> 조립 배송` 흐름을 중심으로 만든다.
- 외부 데이터 수집 결과를 내부 추천 구성과 벤치마크 데이터로 정리한다.
- 고객 화면은 견적 생성, 추천 구성, 게임 성능, 주문 추적에 집중한다.
- 관리자 화면은 주문 확인, 상태 변경, 추천 콘텐츠 관리에 집중한다.

| 영상/방송 작업자 | 용도에 맞는 안정적인 고성능 구성이 필요함 | 용도별 추천 견적과 부품 변경 |
| 운영자/관리자 | 주문과 견적 상태를 빠르게 확인하고 처리해야 함 | 관리자 주문 목록, 상태 변경, 이력 관리 |

## 3. 핵심 사용자 플로우

### 고객 플로우

1. 메인 페이지에서 무료 견적, 추천 구성, 게임 성능, 부품 쇼핑으로 진입
2. 견적 페이지에서 예산, 용도, 선호 게임, 해상도, Windows 옵션 선택
3. 추천된 부품 구성 확인
4. 필요한 경우 부품 변경, 상세 확인, 비교
5. 견적 저장 후 주문 페이지로 이동
6. 배송지, 연락처, 요청사항, 결제 안내 확인
7. 주문 접수
8. 주문번호 또는 마이페이지에서 상태 추적

### 관리자 플로우

1. 관리자 계정으로 로그인
2. 관리자 페이지에서 주문 목록 확인
3. 주문 상세, 주문 항목, 고객 정보, 결제/배송 정보 확인
4. 주문 상태 변경 및 메모 기록
5. 추천글과 추천 구성 관리

### 데이터 운영 플로우

1. Agent가 외부 HTML, 상품, 견적 데이터를 수집
2. 파서가 상품명, 가격, 카테고리, 주요 스펙, CPU/GPU 조합을 정규화
3. DB 동기화 스크립트가 추천 빌드, 벤치마크, 게임 성능 데이터 적재
4. 서버 API가 프론트엔드에 추천 구성과 성능 데이터 제공

- 색상 톤: 어두운 네이비 또는 차콜 배경에 포인트 컬러는 민트/청록 계열

### 노션 블록 추천

- `프로젝트 한 줄 요약`은 Callout 블록으로 변환
- `페이지 속성 추천`은 Table 블록으로 유지
- `구현 목표 체크리스트`는 Todo List 블록으로 유지
- `마일스톤 제안`은 Toggle Heading으로 만들면 접고 펼치기 좋음
- `주요 리스크와 대응`은 Table 블록으로 유지
- `현재까지 개발된 내용`은 3개 컬럼으로 나누면 보기 좋음

### 데이터베이스로 분리하면 좋은 항목

아래 항목은 별도 Notion Database로 만들면 관리가 편합니다.

| DB 이름 | 주요 속성 |
| --- | --- |
| 작업 체크리스트 | 상태, 우선순위, 담당자, 마감일, 영역 |
| 마일스톤 | 상태, 목표일, 완료율, 관련 작업 |
| 리스크 관리 | 위험도, 영향도, 대응 상태, 담당자 |
| 기능 목록 | 영역, MVP 포함 여부, 상태, 관련 페이지 |
| 회의/작업 로그 | 날짜, 작업자, 요약, 다음 액션 |

### 보기 좋게 만드는 팁

- 체크리스트는 `1순위`, `2순위`처럼 우선순위별로 나누면 실행 순서가 명확해집니다.
- 완료된 내용은 너무 길게 펼쳐두지 말고 Toggle 안에 넣는 편이 좋습니다.
- 상단에는 항상 `현재 상태`, `다음 액션`, `막힌 점`만 빠르게 보이게 두는 것이 좋습니다.
- 리스크 표는 빨간색/노란색 태그를 사용해 위험도를 표시하면 관리하기 쉽습니다.
- 마일스톤별 완료율은 Notion Database의 Rollup 또는 Progress Bar 속성을 쓰면 좋습니다.
