# JHSComputer2 기획 정리
> 작성일: 2026-07-03
> 기준: 현재 워크트리 변경사항, `project/docs/PROGRESS.md`, `project/analysis/2026-06-10-claude-improvements.md`

## 1. 서비스 방향

JHSComputer2는 정효성 TV 기반 PC 구매 플랫폼 MVP이다. 사용자는 컴퓨터 부품 지식이 부족해도 예산, 용도, 게임, 해상도 같은 요구사항을 입력해 추천 견적을 받고, 장바구니와 주문 접수까지 이어갈 수 있어야 한다.

핵심 방향은 다음과 같다.

- 부품 쇼핑몰이 아니라 "견적 추천 + 전문가 검수 + 조립 배송" 흐름을 중심으로 설계한다.
- 컴퓨존, 견적왕, 왕가PC 등 외부 데이터 수집 결과를 내부 표준 부품/추천 구성/벤치마크 데이터로 정리한다.
- 고객 화면은 부품 탐색, 추천 견적, 게임 성능, 내 견적, 주문/배송 추적을 제공한다.
- 관리자 화면은 주문 확인, 상태 변경, 추천글/추천 구성 관리, 운영 데이터를 관리하는 방향으로 확장한다.
- MVP에서는 PG 결제보다 주문 접수, 무통장 안내, 관리자 검수, 상태 추적을 우선한다.

## 2. 현재까지 구현된 큰 축

### 2.1 데이터 수집 및 정제

- 컴퓨존 상품 샘플 수집/분석 도구를 작성했다.
- 견적왕 견적 HTML 수집/파싱 도구를 확장했다.
- 왕가PC 상품 수집/파싱/DB 동기화 도구를 추가했다.
- 유튜브 채널 분석 스크립트와 추천글 생성 스크립트를 추가했다.
- CPU/GPU 조합, 게임별 FPS, 추천 빌드, 게임 요구사항 데이터를 `data/`와 `project/samples/`에 누적했다.

### 2.2 DB 및 서버

- MySQL 초기 스키마와 seed SQL을 구성했다.
- 벤치마크용 스키마, 장바구니 스키마, 추천글 seed SQL을 추가했다.
- NestJS API 서버에 인증, 사용자, 부품, 견적, 주문, 장바구니, 벤치마크, AI, 추천 모듈을 구성했다.
- 주문 동기화, 주문번호 추적, 상태 변경 이력, 현재 사용자 조회, 추천 구성 조회, 견적 성능 추정 API 흐름을 추가했다.
- 개발 환경 CORS, JWT 인증, 주문/견적 엔티티 확장을 진행했다.

### 2.3 프론트엔드

- Next.js App Router 기반 고객 웹 화면을 구축했다.
- 메인 랜딩, 로그인, 회원가입, 부품, 견적, 견적 상세, 주문, 주문 상세, 주문 추적, 마이페이지, 장바구니, 포인트, 배송, 추천 견적, 추천 상세, 벤치마크, 관리자 화면을 추가 또는 개편했다.
- `AppShell`을 쇼핑몰형 헤더/푸터 구조로 재작성했다.
- 부품 카드, 부품 비교, 부품 상세 모달, 견적 요약, 추천 빌드 카드, 추천글 컴포넌트, 벤치마크 컴포넌트, 공통 UI 컴포넌트를 분리했다.
- 로컬 저장소 기반 견적/주문 흐름과 서버 API 기반 데이터 로딩을 함께 사용하도록 확장했다.

## 3. 사용자 플로우 기획

### 3.1 고객 플로우

1. 메인 페이지에서 무료 견적, 추천 구성, 게임 성능, 부품 쇼핑으로 진입한다.
2. 견적 페이지에서 예산, 용도, 선호 게임, 해상도, Windows 옵션 등을 선택한다.
3. 추천된 부품 구성을 확인하고 필요한 경우 부품을 변경하거나 비교한다.
4. 견적을 저장하고 주문 페이지로 이동한다.
5. 배송지, 연락처, 요청사항, 결제 안내를 확인한 뒤 주문을 접수한다.
6. 주문번호 또는 마이페이지에서 상태를 추적한다.

### 3.2 관리자 플로우

1. 관리자 계정으로 로그인한다.
2. 관리자 페이지에서 DB 주문과 로컬 주문을 확인한다.
3. 주문 상세, 주문 항목, 고객 정보, 결제/배송 정보를 확인한다.
4. 상태를 변경하고 메모를 남긴다.
5. 추천글과 추천 구성을 관리한다.

### 3.3 데이터 운영 플로우

1. Agent가 외부 사이트 HTML과 상품/견적 데이터를 수집한다.
2. 파서가 상품명, 가격, 부품 카테고리, 주요 스펙, CPU/GPU 조합을 정규화한다.
3. DB 동기화 스크립트가 추천 빌드, 벤치마크, 게임 성능 데이터를 적재한다.
4. 서버 API가 프론트엔드에 추천 구성과 성능 데이터를 제공한다.

## 4. MVP 범위

### 포함

- 고객용 PC 견적 생성 및 추천 구성 조회
- 부품 목록/검색/필터/비교
- 장바구니 및 견적 저장
- 주문 접수와 주문번호 추적
- 관리자 주문 확인 및 상태 변경
- 게임별 성능 참고 데이터
- 추천 견적 콘텐츠와 상세 페이지
- 기본 로그인/회원가입/JWT 인증

### 제외 또는 후순위

- 실결제 PG 연동
- 배송사 실시간 API 연동
- 카카오/구글 소셜 로그인 완성
- 알림톡/문자/이메일 자동 발송
- 재고/가격 실시간 보장
- 커뮤니티, 후기, 댓글, 좋아요

## 5. 주요 리스크

- 실제 DB seed 적용과 API/프론트 E2E 검증이 아직 문서상 완전히 닫히지 않았다.
- 일부 프론트 흐름은 로컬 저장소와 서버 API가 혼재되어 있어 단일 데이터 소스 정리가 필요하다.
- 비밀번호 해시, 관리자 권한 가드, 운영 CORS 제한 등 보안 필수 작업이 남아 있다.
- 외부 수집 데이터의 상품명 정규화, 부품 매칭 정확도, 가격 변동 대응이 서비스 품질에 직접 영향을 준다.
- 추천 견적과 벤치마크 데이터는 출처, 갱신일, 신뢰도 표시 정책이 필요하다.

## 6. 다음 기획 우선순위

1. 고객 주문 전환 플로우를 "견적 생성 -> 견적 저장 -> 주문 접수 -> 상태 추적"으로 고정한다.
2. 로컬 저장소와 서버 저장소의 역할을 명확히 나눈다.
3. 관리자 기능의 최소 운영 단위를 주문 상태 관리와 추천글 관리로 좁힌다.
4. 벤치마크/추천 데이터의 갱신 주기와 표시 정책을 정한다.
5. 배포 전 보안, 환경변수, 검증 체크리스트를 별도 릴리즈 기준으로 관리한다.
